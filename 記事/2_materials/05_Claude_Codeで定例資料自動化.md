# 素材：Claude Codeで毎週の定例資料作成を自動化した話

## 話者

中村さん・30代・コミュニティマネージャー

## 課題

週次定例の資料作成に毎回2〜3時間かかっていた。
Slackのログを読んで、スプレッドシートの数値を確認して、先週との差分をまとめて……という作業の繰り返し。
同じフォーマットなのに毎回ゼロから書き直していた。

## 使ったツール

Claude Code（ターミナルで動かすAIアシスタント）

## やったこと

1. 定例資料のテンプレートをMarkdownファイルで作成
2. Slackのログ取得・スプレッドシートの数値読み込みをClaude Codeに指示
3. 「テンプレートに従って今週の定例資料を作って」コマンド1つで生成

スキルファイル（`/weekly-report`）を作ったので、毎週同じコマンドを打つだけ。

## 結果

資料作成時間：2〜3時間 → 15〜20分に短縮（約85%削減）。
フォーマットの統一性が上がり、「見やすくなった」と言われた。
Slackのログから拾えていなかった情報も可視化されるようになった。

## 話者の発言メモ

「最初はプログラミングが必要だと思って諦めていた。でも全部日本語で指示するだけでよかった」

「コマンド1つで資料のたたき台ができる。あとは数字の確認と追記だけ」

「一度作ったスキルは毎週使える。設定の手間が最初だけなのが良い」

「上司に『効率化したの？』と聞かれて、自慢できるようになりました（笑）」
